# Нейро-помощник Авито (MVP)

Веб-приложение для анализа скриншотов объявления с Avito: общая оценка по шкале 1–10, разбор по критериям, сильные и слабые стороны, короткие рекомендации и сжатый улучшенный текст. Авторизация, база данных и оплата не используются.

Анализ выполняется через **OpenAI Responses API** (не Chat Completions), модели с поддержкой vision: **`gpt-4.1`** или **`gpt-4.1-mini`**. Скриншоты на сервере кодируются в **base64** и передаются в запрос как **`input_image`** (data URL).

Дополнительный контекст для модели задаётся файлом **`app/knowledge/knowledge.txt`** (база знаний по оформлению объявлений).

## Требования

- Python 3.11+
- Ключ API OpenAI

## Переменные окружения

Создайте в корне проекта файл `.env` по образцу `.env.example`:

| Переменная | Описание |
|------------|----------|
| `OPENAI_API_KEY` | Ключ API ([platform.openai.com](https://platform.openai.com/api-keys)) |
| `OPENAI_MODEL` | Необязательно. По умолчанию `gpt-4.1-mini`. Допустимо: `gpt-4.1`, `gpt-4.1-mini` |

Пример:

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4.1-mini
```

## Локальный запуск

1. Клонируйте или скопируйте проект в папку на компьютере.

2. Создайте виртуальное окружение (рекомендуется) и установите зависимости:

```bash
python -m pip install -r requirements.txt
```

3. Создайте файл `.env` и укажите в нём `OPENAI_API_KEY` (и при необходимости `OPENAI_MODEL`).

4. Запустите сервер из **корня проекта** (рядом с папкой `app`):

```bash
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

5. Откройте в браузере: **http://127.0.0.1:8000**

## API

- **`POST /api/analyze`** — `multipart/form-data`
  - **`files`** — от 1 до 10 изображений (JPEG, PNG, WebP, GIF), до **5 МБ** на файл
  - **`niche`**, **`audience`**, **`comment`** — необязательные текстовые поля

При ошибках валидации или недоступности сервиса клиент получает JSON с полем **`detail`** и понятным текстом.

## Структура проекта

- `app/main.py` — приложение FastAPI и раздача фронтенда
- `app/routers/analyze.py` — эндпоинт анализа и проверка лимитов загрузки
- `app/services/openai_service.py` — вызов Responses API и чтение базы знаний
- `app/schemas/analysis.py` — валидация ответа модели (Pydantic)
- `app/knowledge/knowledge.txt` — текст базы знаний
- `static/` — HTML, CSS и скрипт формы
